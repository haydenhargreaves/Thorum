Module editor

structs

EDITOR

Properties

  • version: f32
    • Version of the editor

  • rows: [dynamic]row.ROW
    • Rows of content in the editor at the current time

  • win: ^nc.Window
    • Ncurses window object used for drawing to the screen

  • cur_x: i32
    • Current x position of the cursor (0-indexed, starting from left)

  • cur_y: i32
    • Current y position of the cursor (0-indexed, starting from top)

procedures

editor_init

Create a blank EDITOR struct which is used as the editor state during the lifetime of the program. The rows will be initialized to an array with 0 length and 0 capacity and can be appended to. This procedure will initialize the win object as well. So there is no need to for nil cases elsewhere.

Params

Returns

  • editor.EDITOR
    • The newly created editor with default memory allocated

editor_destroy

Free underlying memory the provided EDITOR struct. This procedure will also free the memory stored in the EDITOR.rows[*].chars. This is a bit of a temporary solution. This procedure will also end the win so there is no need to cleanup the Ncurses window elsewhere.

Params

  • e: ^editor.EDITOR
    • The editor struct you wish to free.

Returns


editor_append_row

Appends a row to the bottom of the rows dynamic array. This will simulation the creation of a newline at the very bottom of the file. Content can be provided to the procedure to create a row with the defined content.

Params

  • e: ^editor.EDITOR
    • The editor struct you wish to append a row to.
  • str: [dynamic]u8
    • Content to place in the newly appended row. Nil can be used to append a row with no content.

Returns


editor_insert_row

Insert a row at pos of the rows dynamic array. This will simulation the creation of a newline at any location in the file. Content can be provided to the procedure to create a row with the defined content. Since an index is used, this procedure will handle out of bounds by returning an error. Negative indexes or indexes above (or equal to) the length will return an error and not insert anything.

Params

  • e: ^editor.EDITOR
    • The editor struct you wish to insert a row into.
  • pos: int
    • The index to insert the row at (0-indexed).
  • str: [dynamic]u8
    • Content to place in the newly inserted row. Nil can be used to append a row with no content.

Returns

  • editor.EDITOR_ERROR
    • If an error occurs, it will be returned. Otherwise, this value will be nil

editor_remove_row

Remove a row from the editor's rows dynamic array. This is used to delete rows from the editor's memory. It does not shrink the capacity of the list, just the length. If the pos provided is below 0 or greater than or equal to the number of rows, an out of bounds error. This procedure will also free the memory that is stored in the row that is deleted. Currently there is no procedure to delete the row, so it will be handled here.

Params

  • e: ^editor.EDITOR
    • The editor struct you wish to remove a row from.
  • pos: int
    • The index to remove the row from (0-indexed).

Returns

  • editor.EDITOR_ERROR
    • If an error occurs, it will be returned. Otherwise, this value will be nil

editor_draw_row

Draw a row onto the screen stored in the editor's window property. If no window is in the state, the procedure will return a missing window error. If an invalid index is provided, below 0 or greater than or equal to the number of rows (calculated from the length of the rows property) then an out of bounds error will be returned. Finally, if the row provided is a nil pointer, the procedure will return an invalid row error.

Params

  • e: ^editor.EDITOR
    • The editor struct that contains the window.
  • row: ^row.ROW
    • The row you wish to draw to the screen.
  • pos: int
    • The column to draw the row at on the window (0-indexed).

Returns

  • editor.EDITOR_ERROR
    • If an error occurs, it will be returned. Otherwise, this value will be nil

editor_move_cursor

Moves the cursor state of the editor passed in. This works by modifying the values cur_x and cur_y. However, this procedure does not actually move the cursor. Just update the state for other procedure to access. This procedure will handle bounds checking and not allow the cursor to overflow.

Params

  • e: ^editor.EDITOR
    • The editor struct that that contains the window.
  • x: i32
    • Number of spaces to move in the x direction (0-indexed, +x is to the right)
  • y: i32
    • Number of spaces to move in the y direction (0-indexed, +y is down)

Returns
